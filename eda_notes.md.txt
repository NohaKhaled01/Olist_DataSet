Table: olist_order_items_dataset
Columns:
order_id	
order_item_id	
product_id	
seller_id	
shipping_limit_date	
price	
freight_value

Contains data for the orders, each with its own order id
Each order has the product id, and the seller id. Plus the price, and the shipping cost [freight value]
The order item id designates the number of items in an order. 
The same order id can be repeated, with the order item id going from 1 to the number of items in that order
Highest number of items in one order = 21 

Worth noting: if an order contains multiple orders for the same product from the same seller,
the shown shipping value is per each item in the order .. which i think is a mispresentation.
if i need to calculate the shipping costs, i need to group the product and seller ids per order before i do that.

Shipping limit date -- not sure if this is the deadline for the seller to hand the order to the carrier, or it is the actual shipping date.
